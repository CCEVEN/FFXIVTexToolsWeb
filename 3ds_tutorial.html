3DS Tutorial

In this tutorial I will be combining 2 different models into 1. I will go over:
  • The Tools Needed
  • Exporting 3D models from TexTools
  • Importing 3D models into 3DS Max
  • Combining the 3D models
  • Saving the models in the correct format
  • Importing the models back into TexTools


    • I will be modifying the Acolyte's Skirt, begginging with the original(Left) and ending with the modified version (Right)

    Tools Needed

  • 3DS Max
  • OpenCollada Plugin

    • Autodesk 3DS Max: https://www.autodesk.com/products/3ds-max/overview
    or
    [Free]Autodesk 3DS Max Student: https://www.autodesk.com/education/free-software/3ds-max

    OpenCOLLADA Plugin: https://github.com/KhronosGroup/OpenCOLLADA/releases

    On the Github website select your Version of 3ds Max to download the zip file.


    The instuctions for installing the plugin are within the downloaded zip file. Essentailly you copy the COLLADAMaxNew.dle file in Release_max[Year] into your 3dsMax/plugins folder.

    Rigging_CombineSkin MaxScript: https://github.com/liinko/FFXIVTexToolsWeb/blob/master/Rigging_CombineSkin.zip?raw=true

    Note:In this tutorial I will be using 3DS Max 2017, but older/newer versions should work.

    Exporting Models

    The first thing we have to do is extract the models we will be working with in 3ds max.

    In this case I will extract:
  • Acolyte's Skirt (Hyur Midlander Female)
  • Bridesmaid's Tights (Hyur Midlander Female)

    • Importing into 3DS Max

    After we have 3DS MAX set up and the models have been extracted we will be importing them into 3ds max.
    Once all meshes are imported, make sure to save often to avoid any lost work.

    Go to the 3ds max menu and select import:


    We next change the "Files of type" to "OpenCollada (*.DAE)



    We always want to open the piece of equipment that will go back into the game first.
    In this case I will open Acolyte's Skirt first, so I navigate to [TexTools Folder]\Saved\Legs\Actolyte's Skirt\3D\
    Then select the file c0201e0120_dwn.DAE and open.

    The following items should show up in your Scene Explorer



    We will not be modifying any bones so they can be collapsed.


    We then import the next item using the same proccess, in this case the bridesmaids tights

    You will always get a "Import Name Conflict" warning, select Rename all and OK


    The Name Conflict occurs because both of the models use the same bones.


    Now you should see both meshes overlayed ontop of one another in the Viewports. I do the following to the bottom right viewport for better visability.

    Deleting unwanted parts

    Now that we have both meshes imported its time to delete the parts we do not want, and merge the parts we do.

    Selecting a part in the Scene Explorer will highlight the part in the viewports.


    Note: keep in mind which group contains the body mesh and which contains the equipment mesh.


    In this case I've selected c0201e0120_dwn_0.1, which is the boot like part of the Acolyte's Skirt, which I do not want in the final product.
    So after selecting the part just hit your delete button, and the mesh should be gone from the group and the change reflected in the viewport.


    We do the same to the top equipment and leg portions of the bridesmaids tights, which we do not want.


    And finally we do the same to some the leg portion of the Acolytes skirt which is in the same spot as part of the tights.

    Merging and moving meshes

    Now we need to add the Bridesmaids Tights mesh to the Acolyte Skirt mesh, essentailly replacing the boot like part we deleted earlier.

    We have a bit of a problem though, we deleted only 1 mesh part, but the bridesmaids tights are 3 meshes.



    So what we'll have to do is merge the 3 parts of the bridesmaids tights into 1. This is where the script you downloaded comes in.

    With the 3 meshes we want to combine selected, go to Scripting > Run Script


    Then we select Combine Skin


    When the script finishes we are left with the combined mesh.


    Now we can drag and drop the mesh into the group where the mesh being replaced is.


    Then rename it to the name of the mesh it is replacing by either right click > rename, or clicking on the name again once it is selected.

    Changing texture coordinates.

    Because we had a mesh from the bridesmaids tights brought into the acolytes skirt mesh, the texture coordinates for the bridesmaids tights are now going to fall in the acolytes skirt texture.

    To see this we select all of the meshes we now have for group_0.


    We then add Unwrap UVW to the modifier list and select Open UV Editor...


    Here we can see where all of the meshes are going to be mapped to on the texture.
    To line the texture up correctly to the mesh, select the 4 squares, and then select a texture for the background (I prefer Normal map for a more detailed view).


    After that is done you can close the UV Editor, select only the mesh we're interested in.


    Go back to the modifier list, select Unwrap UVW and Open UV Editor ... again.


    You should now be able to see where the bridesmaids tights mesh we brought over will get its texture from
    From here you can select the meshes UVs and move/rotate/scale them and place them wherever you think is best
    Keep in mind that where you place it, you will most likely have to modify the items textures to change/color that specific part of the mesh.

    You can go back to having them all selected, and open the UV editor to check whether they are still overlapping, or verify placement.
    Here's what mine ended up looking like.


    Now because the order in which things appear in the modifier does matter when saving, we need to go to the modifier list of each meshe we selected for Unwrap UV and move it beneath Skin.
    Do this for each of the meshes that were selected or TexTools will not successfuly import.

    Exporting

    Now that we have the desired look we need to export the file in collada format.

    Go to the menu > Export.


    Change the "Save as type:" to OpenCOLLADA (*.DAE).


    Make sure to save to the same directory, in this case [TexTools Folder]\Saved\Legs\Actolyte's Skirt\3D\


    Select the file we are going to replace and select Save.


    When asked if you want to replace the existing file, select Yes.

    You will then see the export settings, make sure they are as follows

    Correcting the file order

    Because we moved the mesh from one group to another, the order in which they are written in the .dae file is most likely incorrect.
    This only needs to be done if there is more than 1 mesh in the group, and the mesh replaced was the primary mesh "_0", "_1", etc. and not a mesh part "_0.1", "_0.2", etc.
    In the case of this tutorial we don't have to, but I'm placing this here in case it is needed.

    We need to do this because as of now, TexTools does not have a way to handle out of order meshes that begin with a mesh part "_0.1, "_0.2", etc., and I don't know how to correct the order in 3ds max.

    I know this part is a pain, and I'll try and add a fix for it within TexTools for the next release.

    To correct this, we have to open the .dae file in a text editor (I use Notepad++) and copy paste the section into the correct place.

    Go to the section labled library_geometries


    Every mesh part has a geometry section, and they should be in ascending order numerically, starting with the mesh without a "." in it in thsi case "geom-c0201e0120_dwn_0" should be the first.
    However the first section is _0.1, which happens when you delete a mesh and then replace it with another again later in 3ds.

    So what we do is go to where the geometry section _0 is located, and cut(ctrl+x) the entire geometry section from <geometry> to </geomerty>
    Note:The entire section goes down further than the screenshot.


    Then pase it (ctrl+v) at the top of the section right after <library_geometries> and before the first <geometry>


    Now we have to do the same thing for the <controller> section of _0 located in the <library_controllers> section


    Placing it at the top of the section between the <library_controllers> section and the fisrt <controller>


    Then Save the file, and we are done.
